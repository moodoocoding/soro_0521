## 1. 지금까지 진행된 작업 (Work Completed)

### 🎨 픽셀 아트 에디터 레이아웃 리팩토링 및 CSS 수정
- **문제점**: 에디터 레이아웃(`.pixel-editor-layout`)에 `position: fixed`가 적용되어 있어, `transform` 스타일이 적용된 부모 요소(`.drawer-content`) 내부에서 뷰포트 기준이 아닌 부모 요소 기준으로 레이아웃이 깨지고 위치가 어긋나던 현상을 발견했습니다.
- **수정 사항**:
  - `position: fixed`를 `position: absolute`로 변경하고 크기를 `100vw/100vh` 대신 부모 크기에 맞춘 `100%/100%`로 설정했습니다.
  - 에디터 하위 요소들이 겹치지 않도록 그리드 배치(`grid-column`, `grid-row`)를 명시적으로 할당했습니다. (예: Action bar를 `grid-column: 2 / 4`에서 `grid-column: 2; grid-row: 2` 등으로 배치 조정)
  - 모바일 해상도에서 뷰포트에 고정되어 깨지던 문제를 부모 요소 상대 크기로 맞추어 개선했습니다.

### 🔌 초기 픽셀아트 강제 전체화면 로직 제거 (`app.js`)
- **문제점**: 픽셀아트 공모전 클릭 시 상세 안내(대회 안내)를 보여주지 않고 즉시 전체화면 에디터 모드로 전환되어 레이아웃이 깨지고 빈 화면이 노출되는 문제가 있었습니다. (비동기 데이터 로딩 완료 전에 에디터 DOM을 찾으려 하여 오동작)
- **수정 사항**:
  - `openContestDetails` 내부에서 호출되던 픽셀아트 전용 즉시 전체화면 설정 코드를 제거했습니다.
  - 이제 다른 공모전과 마찬가지로 상세 안내 화면이 먼저 로출됩니다.

## 2. 현재 발생하고 있는 이슈 분석 (Current Issues)

### ⚠️ 이슈 1. 왜 마음대로 '작품 접수하기' 탭/화면으로 접속되는가?
- **원인**:
  - `openContestDetails` 함수 하단에 `if (contest.id !== "pixelart" || !currentUser)` 조건식으로 인해, 픽셀아트 대회이면서 로그인 상태인 경우 `switchDrawerTab("guide")`가 호출되지 않아 탭 상태가 제대로 초기화되지 않는 상태였습니다.
  - 또한, `app.js` 하단에 레이아웃 디버깅용 코드로 작성된 `forceOpenPixelArt()`가 남아있어 페이지가 새로고침될 때마다 자동으로 픽셀아트 대회 창이 강제로 열리고 있습니다.

### ⚠️ 이슈 2. 픽셀아트를 클릭할 때 빈 화면 또는 잘못된 로딩이 발생하는 문제
- **원인**:
  - 픽셀아트 에디터의 컨테이너인 `#pixel-draw-container`가 기본적으로 숨겨져 있지 않고 (`display: none`이 없음), HTML에 바로 포함되어 있어 서랍을 열자마자 화면을 가리거나 정상적이지 않은 상태로 렌더링을 방해하고 있습니다.
  - 사용자가 "직접 도트 찍기" 버튼을 누르기 전까지는 에디터 영역이 비활성화(숨김) 상태여야 합니다.

## 3. 앞으로 해결할 작업 계획 (Future Action Plan)

### 1단계: 임시 디버깅 코드 제거
- `app.js` 맨 아래에 있는 `forceOpenPixelArt()` 함수 및 DOMContentLoaded 이벤트 리스너 제거하기.

### 2단계: 픽셀아트 진입 시 탭 초기화 및 기본 뷰 수정
- `openContestDetails` 함수 내에서 픽셀아트 여부와 상관없이 **항상** `switchDrawerTab("guide")`를 호출하여 첫 진입 시 대회 안내가 먼저 보이도록 수정.
- `setupDynamicFormFields`에서 픽셀아트 에디터 영역인 `#pixel-draw-container`를 기본적으로 `display: none;` 상태로 렌더링하고, 대신 **파일 업로드 폼** 또는 **제출 방법 선택 화면**이 먼저 보이도록 변경.
- 사용자가 "직접 도트 찍기" 버튼을 클릭할 때에만 에디터를 `display: grid`로 표시하고 전체화면 모드(`drawer-fullscreen` 클래스 추가 및 주변 요소 숨김)를 활성화하도록 전환 로직 일치화.

### 3단계: 로컬 테스트 및 서브밋 흐름 검증
- 수정 후 로컬 서버(`http://localhost:3000`) 환경에서 다음 흐름을 수동 검증:
  1. 공모전 목록에서 "픽셀아트 공모전" 클릭 -> 대회 안내 탭 정상 노출 확인.
  2. "직접 도트 찍기" 클릭 -> 전체화면 에디터가 정상 비율로 뜨는지 확인.
  3. 도트 그리기 및 도구 작동 여부 확인.
  4. "임시 저장" 및 "최종 제출" 기능 정상 작동 테스트.
